# Gridea 项目说明与技术架构

## 项目简介
- Gridea 是一款桌面端静态博客写作客户端，支持 Markdown 写作、主题定制、预览与一键发布。
- 应用基于 Electron 主进程 + Vue 2 渲染进程，使用 TypeScript 与 `lowdb` 本地存储。
- 支持发布到 GitHub Pages、Coding Pages、Gitee Pages、SFTP 与 Netlify。

## 技术栈
- 桌面框架：Electron（`src/background.ts:25`）
- 前端：Vue 2、TypeScript、Ant Design Vue、Vue Router、Vuex（`src/main.ts:43`，`src/router.ts:24`，`src/store/index.ts:7`）
- 预览服务：Express（`src/server.ts:3`）
- 渲染引擎：EJS 模板 + Less 编译（`src/server/renderer.ts:441`，`src/server/renderer.ts:502`）
- 数据存储：lowdb（`src/server/model.ts:29`、`src/server/model.ts:33`、`src/server/model.ts:37`）
- Git 发布：isomorphic-git（`src/server/deploy.ts:7`）
- 其他：Sentry、Prism、KaTeX、hpagent（代理）

## 事件与进程通信
- 站点加载
  - 渲染进程触发 `app-site-reload`，主进程加载 Posts/Tags/Menus/Theme/Setting 并返回 `app-site-loaded`（`src/server/events/site.ts:13–21`）。
- 预览渲染
  - 渲染进程触发 `html-render`，主进程执行 `Renderer.preview()` 并返回 `html-rendered`（`src/server/events/renderer.ts:10–16`、`src/server/renderer.ts:51–54`）。
- 发布
  - 渲染进程触发 `site-publish`，主进程先渲染整站，再根据平台执行发布并返回结果（`src/server/events/deploy.ts:16–40`）。
